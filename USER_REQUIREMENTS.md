## 已确认的要求（持续维护；每次开发前先读）

### Git / 发布流程
1. 每次更改完成后：**提交 commit，并推送到 GitHub**。

### 财报（finreport_fetcher）输出格式
2. 财务报表中**每一项都必须有对应的 `key`**（用于模板/运算引用；即使是未映射科目，也要自动生成 key）。
3. 科目展示中英对照规则：**括号内只显示英文翻译；没有英文就不加括号**（例如：`营业收入 (Operating revenue)`；未知项保持 `中文`）。
4. 输出 Excel **不要**包含 `科目_CN`、`科目_EN` 两列（避免重复）。

### Excel 可读性/排版
5. `数值` 列右侧需要保留一定空隙（例如在数值列右侧插入空白列/调整列宽），方便肉眼查看数字。
6. 科目列不同类别的缩进需要保留（分组标题 + 子项缩进）。
7. 每列宽度要自适应内容（autofit），不要出现明显挤压/遮挡。

### 提取行为
8. 输出目录按公司分组：同一公司的 xlsx+pdf 放入同一目录：`{公司名}_{公司代码}`，其中 PDF 与 XLSX 同目录（不使用 `pdf/` 子目录）。
9. 每次提取前的清理：**只删除本次提取公司的历史文件**（同 code6 的 xlsx/pdf），不删除其他公司数据。

### 图表模板/计算
10. 图表 transform 口径支持：单期(单季) / 累计(YTD) / TTM，并可在模板中使用。
11. 图表只通过模板执行：模板为 TOML，每个模板一个文件；支持 `--template <file>` 与 `--templates <dir>` 两种运行方式。
12. 图表输出目录：`output/{公司名}_{公司代码}/charts/`。

### 交付 / 测试流程
13. 每次更新完程序后：**将整个项目同步拷贝到** `/mnt/hgfs/share_with_vm/` 下的一份（用于用户测试）。
14. 每次更改完程序后：我需要**自己先做一次冒烟测试**（至少：fetch 1 个股票 + 检查输出 Excel 列 + 可选跑 1 个图表命令）。
15. 每次更改完成后：更新必要文档（README + docs 中与本次改动相关的说明）。
